centos登录失败到修复笔记

发表于 2018-06-26 更新于 2020-03-31 分类于 centos 阅读次数：

一不小心删除了~/.ssh/authorized_keys，导致 ssh 登录始终报Permission denied (publickey,gssapi-keyex,gssapi-with-mic).，然后通过腾讯云 web 控制台重新加载密钥，仍旧不能解决问题；
尝试重置密码，通过网页自带登录窗口报服务器健全失败，最后在登录页面的其他登录方式中选择VNC，输入刚才重设的账户和密码登录成功。

登录成功之后，查看/root/.ssh 下并不包含authorized_keys文件，即使之前通过重新下发密钥也没有重新生成。通过VNC的终端，手动创建了authorized_keys文件，并设置权限为 600，然后将另外一个服务器中的authorized_keys的内容拷贝过来，然后尝试本机重新登录成功。

发表于 2018-06-14 更新于 2020-03-31 分类于 SpringBoot 阅读次数：

  • 约定优于配置（convention over configuration），也称作按约定编程，是一种软件设计范式，旨在减少软件开发人员需做决定的数量，获得简单的好处，而又不失灵活性。
    本质是说，开发人员仅需规定应用中不符约定的部分。例如，如果模型中有个名为 User 的类，那么数据库中对应的表就会默认命名为 user。只有在偏离这一约定时，例如将该表命名为”user_info”，才需写有关这个名字的配置。

  • Jenkins 就是专门来负责如何将代码变成可执行的程序包，将它部署到目标服务器中，并对其运营状态（日志）进行监控的软件。自动化、性能、打包、部署、发布、发布结果自动化验证、接口测试、单元测试等等关于我们打包测试部署的方方面面 Jenkins 都可以很友好的支持。

SpringCloud学习笔记

发表于 2018-06-08 更新于 2020-03-31 分类于 java 阅读次数：

  • Spring Cloud 其实是一套规范，而不是一个可以拿来即用的框架，而 Spring Cloud Netflix 以及 Spring Cloud Consul，才是 Spring Cloud 规范的实现。当然了，从今天开始，Spring Cloud 的实现又多了一种，Spring Cloud Alibaba，-_-。

  • Spring Cloud 是一个基于 Spring Boot 实现的云应用开发工具，它为基于 JVM 的云应用开发中的配置管理、服务发现、断路器、智能路由、微代理、控制总线、全局锁、决策竞选、分布式会话和集群状态管理等操作提供了一种简单的开发方式。

  • Spring Cloud 包含了多个子项目（针对分布式系统中涉及的多个不同开源产品），比如：Spring Cloud Config、Spring Cloud Netflix、Spring Cloud CloudFoundry、Spring Cloud AWS、Spring Cloud Security、Spring Cloud Commons、Spring Cloud Zookeeper、Spring Cloud CLI 等项目。

  • Spring Cloud Netflix 项目是 Spring Cloud 的子项目之一，主要内容是对 Netflix 公司一系列开源产品的包装，它为 Spring Boot 应用提供了自配置的 Netflix OSS 整合。通过一些简单的注解，开发者就可以快速的在应用中配置一下常用模块并构建庞大的分布式系统。它主要提供的模块包括：服务发现（Eureka），断路器（Hystrix），智能路由（Zuul），客户端负载均衡（Ribbon）等。

  • 我们使用 Spring Cloud Netflix 中的 Eureka 实现了服务注册中心以及服务注册与发现；而服务间通过 Ribbon 或 Feign 实现服务的消费以及均衡负载；通过 Spring Cloud Config 实现了应用多环境的外部化配置以及版本管理。为了使得服务集群更为健壮，使用 Hystrix 的融断机制来避免在微服务架构中个别服务出现异常时引起的故障蔓延。
