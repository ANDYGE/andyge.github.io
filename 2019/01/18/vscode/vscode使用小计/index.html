1.2. 配置文件 setting.json

没安装一个插件，都会有一些配置节点加入，如安装完 Markdown All In One 之后，会加入 markdown.extension.\* 类别的配置节点。

可以通过这些配置改写默认的行为。

2. 常用插件

2.1. Prettier

Prettier 是一个比较武断的格式化工具，到目前为止，如果只写 markdown 的话，只安装一个 Prettier 就够了。Prettier 只关注格式化，并不具有 lint 检查语法等能力，所以开始大家可能会经常碰到 lint 搭配 prettier 会出现冲突。

  • 能够格式化 markdown 表格
  • 能够格式化满足 markdownlint 要求的样式，不符合要求的可以通过 .markdownlint.json 忽略。
  • 中英混合，自动在英文单词两边添加空格

2.1.1. .prettierrc

放置在根目录下的配置文件是能够生效的。

如果在 .prettierrc 中禁用了 tab ，这时候 .editorconfig 中的配置将会生效。

2.2. Markdown All In One

  • 表格格式化
  • 列表助手
  • 如果不装 editorconfig 插件，它的格式化快捷键会覆盖 Prettier 的（不确定）

2.3. EditorConfig for VS Code

强烈推荐安装，如果没安装如 Prettier 插件，将会采用项目目录下的 .editorconfig 的配置。

2.4. Markdown Preview Enhanced

能够支持绝大多数的效果，如数学公式，各种流程图等冷僻的 Markdown 标签。
