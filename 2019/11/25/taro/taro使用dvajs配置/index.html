1.10. bug 列表

1.10.1. undefined is not a function (evaluating 'app.\_models[typeof Symbol === "function" ? Symbol.iterator : "@@iterator"]()')

这个问题出现在安卓的模拟器上，解决办法就是通过ctrl+m，设置dev setting，将 debuger server 地址和端口设定为当前本机 IP 地址。

1.10.2. 安卓和苹果模拟器上仅需启动live Reload即可

不需要关注Hot Reloading，开启该项会有问题。

1.10.3. Cannot add a child that doesn't have a YogaNode to a parent without a measure function! (Trying to add a 'RCTRawText [text: ]' to a 'RCTView')

导致该错误原因是在 RCTView 组件中存在空的字符串。

不过也有其他可能的情况：

  1. 在组件代码中添加了注释
  2. 标签包裹次序出现了问题
  3. map 函数写法有问题
  4. 三目运算符中将空字符串“”改成<View></View>

1.10.4. taro转微信小程序出现 TypeError: _index2.default.RefsArray is not a constructor错误

1.10.5. 引入的组件命名一定要大写开头

1.10.6. 保持 @tarojs/cli 的版本与各端依赖版本一致

在使用 Taro 进行多端开发中，请保持 Taro CLI 的版本与你项目的依赖版本一致，否则可能会出现编译错误或者运行时错误。可以通过如下命令：

1.10.7. 安卓调试跨域问题

在安卓模拟器的debug server host & port中，设置为localhost:8081，本地yarn dev:rn启动后，打开模拟器的remoting js debugging。此时会打开浏览器窗口http://localhost:8081/debugger-ui/。
